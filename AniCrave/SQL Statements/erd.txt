AniCrave Database Entity Relationship Diagram (ERD)

[ USERS ] (Users)
--------------------------------------
PK  id              : INT (Unique User ID)
    username        : VARCHAR (Unique login name)
    email           : VARCHAR (Unique email address)
    password_hashed : VARCHAR (Secured password)
    gender          : VARCHAR (User gender)
    birthday        : DATE (User birthdate)
    contact         : VARCHAR (Phone/Contact number)
    bio             : TEXT (User biography)
    profile_picture : VARCHAR (Path to avatar)
    created_at      : TIMESTAMP (Account creation time)

      |
      | 1:N (A user has many favorites)
      v

[ FAVORITES ] (Junction Table)
--------------------------------------
PK  id              : INT (Favorite ID)
FK  user_id         : INT (References USERS.id)
FK  anime_id        : INT (References ANIME.id)
    anime_title     : VARCHAR (Denormalized title)
    added_at        : TIMESTAMP (Timestamp added)

      ^
      | 1:N (An anime is favorited by many users)
      |

[ ANIME ] (Content)
--------------------------------------
PK  id              : INT (Unique Anime ID)
    title           : VARCHAR (Anime name)
    cover_image     : VARCHAR (Portrait image URL)
    banner_image    : VARCHAR (Landscape image URL)
    description     : TEXT (Synopsis/Summary)
    genres          : VARCHAR (List of categories)
    year            : INT (Release year)
    season          : VARCHAR (Release season)
    type            : VARCHAR (Format: TV, Movie)
    status          : ENUM (Airing status)

      |
      | 1:N (An anime has many characters)
      v

[ CHARACTERS ] (Cast)
--------------------------------------
PK  id              : INT (Character ID)
FK  anime_id        : INT (References ANIME.id)
    name            : VARCHAR (Character name)
    char_role       : VARCHAR (Character role: Main/Supp)
    image_url       : VARCHAR (Portrait image URL)
